## git reflog 

查看merge操作的上一个提交记录的版本号

## git reset --hard 版本号

reset到merge前一次记录
	
## git merge --abort

终止本次操作，之后就可以正常update了，抢救完成

## 如果已经push了呢？

那就要看权限够不够了，执行完上面三条指令后，git push origin <分支名> --force，将本地代码覆盖远端代码
